Requirements – Payment Gateway System

FR-01: Support multiple payment gateway providers (e.g., Paytm, Razorpay).

FR-02: Allow easy addition of new payment gateway providers without modifying existing business logic.

FR-03: Provide a standard payment processing flow including validation, initiation, processing, and confirmation.

FR-04: Handle payment errors gracefully with appropriate exception handling and user feedback.

FR-05: Support automatic retry for transient failures such as network timeouts or temporary gateway unavailability.
